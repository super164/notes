# HTTP 常用状态码详解

HTTP 状态码是服务器对客户端请求的**响应状态标识**，由 3 位数字组成，首位数字定义了状态码的类别，共分 5 类；日常开发 / 运维中高频使用的是**2xx 成功、3xx 重定向、4xx 客户端错误、5xx 服务端错误**四类，下面重点讲解你提到的 200/201/301/302/400/404/500，同时补充同类别高频常用状态码，方便一站式掌握。

## 状态码整体分类（首位数字含义）

|    类别    | 首位数字 |                           核心含义                           |
| :--------: | :------: | :----------------------------------------------------------: |
|   信息性   |   1xx    |         请求已接收，继续处理（临时响应，日常极少用）         |
|    成功    |   2xx    |                  请求已成功接收、理解、处理                  |
|   重定向   |   3xx    |            需客户端进一步操作（跳转）才能完成请求            |
| 客户端错误 |   4xx    | 客户端请求有错误（语法 / 权限 / 资源不存在等），服务器无法处理 |
| 服务端错误 |   5xx    |        服务器接收到有效请求，但自身处理过程中发生错误        |

- **核心分类**：2xx 成功 | 3xx 重定向 | 4xx 客户端错误 | 5xx 服务端错误

  ## 一、2xx 成功（请求正常处理）

  | 状态码 |         含义         |             核心使用场景             |             关键说明              |
  | :----: | :------------------: | :----------------------------------: | :-------------------------------: |
  |  200   |      OK（成功）      | 所有正常请求、查询数据、获取静态资源 |  最常用，返回请求对应数据 / 资源  |
  |  201   |  Created（已创建）   |  新增资源（创建用户 / 订单 / 文件）  | 响应头带 Location，指向新资源 URL |
  |  204   | No Content（无内容） |   删除资源、无感更新、无需返回数据   |     响应体为空，仅返回响应头      |

  ## 二、3xx 重定向（需客户端跳转完成请求）

  | 状态码 |          含义          |           核心使用场景           |               关键说明               |
  | :----: | :--------------------: | :------------------------------: | :----------------------------------: |
  |  301   |       永久重定向       | 域名更换、页面永久迁移、SEO 优化 |     浏览器缓存跳转关系，权重转移     |
  |  302   |       临时重定向       |    未登录跳登录、临时页面维护    | 不缓存，可能篡改请求方法（POST→GET） |
  |  304   |   未修改（协商缓存）   |  静态资源（图片 / JS/CSS）缓存   |    客户端用本地缓存，提升加载速度    |
  |  307   | 临时重定向（保留方法） |      替代 302，业务临时跳转      |    严格保留原请求方法，无篡改风险    |

  ## 三、4xx 客户端错误（请求本身有问题，服务器无法处理）

  | 状态码 |      含义      |                     核心场景 + 排查要点                      |
  | :----: | :------------: | :----------------------------------------------------------: |
  |  400   |    错误请求    | 参数非法 / 格式错误、JSON 解析失败、请求头异常 → 检查前端传参 / 请求格式 |
  |  401   |     未授权     |      未登录、Token 过期 / 无效 → 校验登录状态、认证信息      |
  |  403   |    禁止访问    |      已登录但无权限、IP 被封禁 → 检查用户角色、权限配置      |
  |  404   |   资源不存在   | URL 错误、接口 / 页面未部署、资源已删除 → 核对请求路径、服务器部署情况 |
  |  405   | 请求方法不允许 | 接口仅支持 GET 却用 POST、方法不匹配 → 检查请求方法与接口定义是否一致 |
  |  413   |   请求体过大   | 上传文件超限制、请求参数过多 → 检查服务器配置、前端提交大小  |

  ## 四、5xx 服务端错误（请求有效，服务器处理出错）

  | 状态码 |      含义      |                     核心场景 + 排查要点                      |
  | :----: | :------------: | :----------------------------------------------------------: |
  |  500   | 服务器内部错误 | 代码异常（空指针 / 越界）、数据库连接失败、依赖服务挂掉 → 查看服务器日志定位代码问题 |
  |  502   |    网关错误    | Nginx 反向代理后端服务挂掉 / 无响应 → 检查后端服务是否启动、端口是否正确 |
  |  503   |   服务不可用   | 服务器维护、资源耗尽（CPU / 内存满）→ 查看服务器状态、是否高峰期 |
  |  504   |    网关超时    | 后端处理过久（复杂查询 / 接口调用）、超时配置过短 → 优化后端逻辑、调整网关超时时间 |

  ## 五、核心速记口诀（快速回忆）

  200 成功 201 创，301 永久 302 跳，304 缓存别忘掉；

  400 参数 401 登，403 无权 404 找不到；

  500 服务内部错，502 网关挂了机，503 维护 504 超时。

  ## 六、排错通用技巧

  1. 先看状态码首位：2→成功，3→跳转，4→前端问题，5→后端问题；
  2. 前端排查：F12 Network 面板，检查 URL、传参、请求方法；
  3. 后端排查：4xx 盯请求，5xx 查日志，3xx 看跳转地址。
