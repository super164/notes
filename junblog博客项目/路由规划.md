路由规划

- 我按你这个个人博客项目，给你拆成 公开接口 、 登录用户接口 、 管理员接口 三层
- 这样你后面写路由、写中间件、写前端联调都会很清晰
公开接口

- 说明：任何人都能访问，不挂 JWT
- 适合“浏览内容”的能力
- POST /api/login 作用：用户登录，返回 token
- GET /api/articles 作用：前台文章列表，支持分页、分类筛选、标签筛选
- GET /api/articles/:id 作用：前台文章详情
- GET /api/categories 作用：前台分类列表
- GET /api/tags 作用：前台标签列表
- GET /api/articles/:id/comments 作用：获取某篇文章的评论列表
- GET /api/site/profile 作用：博客站点作者信息，比如昵称、简介、头像、社交链接
- GET /api/site/stats 作用：站点统计信息，比如文章数、分类数、阅读量
登录用户接口

- 说明：需要登录，挂 JWT 中间件
- 适合“当前用户参与互动”的能力
- GET /api/profile 作用：获取当前登录用户信息
- PUT /api/profile 作用：修改当前登录用户资料
- PUT /api/profile/password 作用：修改密码
- POST /api/comments 作用：发表评论
- POST /api/comments/:id/reply 作用：回复评论
- POST /api/articles/:id/like 作用：点赞文章
- DELETE /api/articles/:id/like 作用：取消点赞
- POST /api/articles/:id/favorite 作用：收藏文章
- DELETE /api/articles/:id/favorite 作用：取消收藏
- GET /api/my/comments 作用：查看我发过的评论
- GET /api/my/favorites 作用：查看我收藏的文章
管理员接口

- 说明：先挂 JWT，再校验 role == admin
- 适合“后台管理”的能力
- GET /api/admin/profile 作用：获取当前管理员信息
- GET /api/admin/dashboard 作用：后台首页统计数据
- POST /api/admin/articles 作用：新增文章
- GET /api/admin/articles 作用：后台文章列表
- GET /api/admin/articles/:id 作用：后台文章详情
- PUT /api/admin/articles/:id 作用：更新文章
- DELETE /api/admin/articles/:id 作用：删除文章
- POST /api/admin/categories 作用：新增分类
- GET /api/admin/categories 作用：后台分类列表
- PUT /api/admin/categories/:id 作用：更新分类
- DELETE /api/admin/categories/:id 作用：删除分类
- POST /api/admin/tags 作用：新增标签
- GET /api/admin/tags 作用：后台标签列表
- PUT /api/admin/tags/:id 作用：更新标签
- DELETE /api/admin/tags/:id 作用：删除标签
- GET /api/admin/comments 作用：后台评论列表，可按状态筛选
- PUT /api/admin/comments/:id/status 作用：审核评论状态
- DELETE /api/admin/comments/:id 作用：删除评论
- POST /api/admin/upload 作用：上传文章封面或正文图片

```go
  /articles/:id/comments

  /admin/comments
  /admin/upload
```



中间件挂载建议

- 公开接口组：不挂 JWT
- 用户接口组：挂 JWTAuth()
- 管理员接口组：挂 JWTAuth() ，再加一个管理员校验
逻辑上建议这样分：

```
api := r.Group("/api")

public := api.Group("/")
user := api.Group("/")
admin := api.Group("/admin")
```
然后：

- public 不加中间件
- user.Use(JWTAuth())
- admin.Use(JWTAuth(), AdminOnly())



你当前项目最建议先实现的接口顺序

- 第一批：
  - POST /api/login
  - GET /api/profile
- 第二批：
  - POST /api/admin/categories
  - GET /api/admin/categories
- 第三批：
  - POST /api/admin/articles
  - GET /api/admin/articles
  - GET /api/articles
  - GET /api/articles/:id
- 第四批：
  - POST /api/comments
  - GET /api/articles/:id/comments
  - GET /api/admin/comments
- 第五批：
  - POST /api/admin/upload
